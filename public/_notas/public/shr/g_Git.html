Fluxo de trabalho

  • O Git é um sistema de controle de versão.
  • Permite manter uma colecção de arquivos organizada e realizar operações como mescla de arquivos, recuperação de versões anteriores, etc
  • Seu repositório local consiste em três "árvores" (trees) mantidas pelo Git.
  • A primeira é o seu Diretório de Trabalho, que contém os arquivos reais.
  • A segunda é o Índice (index), que atua como uma área de preparação.
  • A terceira é a HEAD, que aponta para o último commit que você fez.

Criando um repositório vazio

  • comando: git init
  • O Git necessita de uma pasta vazia para funcionar.
  • Crie uma pasta no local onde deseja trabalhar.
  • Execute o comando ao lado para iniciar o repositório.

    # inicia o repositório na pasta atual
    git init

Clonando um repositório

  • comando: git clone /caminho/para/o/repositorio.
  • Podemos clonar um repositório já existente local ou na nuvem.
  • Este repositório pode ter arquivos prévios ou estar vazio.
  • Você deve ter permissão para ler o repositório ou o mesmo ser público.
  • Execute o comando ao lado para clonar o repositório.

    # clona um repositório hospedado no github
    git clone https://github.com/mwar-br/nasa.git

Crie os arquivos no diretório de trabalho.

Adicionar arquivos ao monitoramento Git

  • Depois de clonar ou criar o repositório a pasta passa a ser mapeada.
  • O Git consegue saber quando um arquivo foi criado, alterado ou excluído.
  • Isso pode ser checado com o comando git status
  • Quando existe uma alteração no arquivo, o comando git add permite registrar esta alteração. 

    # checa se houve alterações na pasta
    git status

    # adiciona todos os arquivos ao diretório de Trabalho
    git add .

    # adiciona apenas o arquivo mencionado ao diretório de Trabalho
    git add texto.txt

    # adiciona todos os arquivos do tipo txt ao diretório de Trabalho
    git add *.txt

Registrar as alterações: Commit

  • Quando finalizamos a edição de um arquivo e queremos registrar essa alteração usamos o comando commit.
  • Ele nos ajuda a criar um histórico das alterações que foram realizadas no código.
  • Inclusive podendo desfazê-las posteriormente.

    # checa se houve alteração na pasta
    git status

    # Adicionamos todos os arquivos modificados no diretório de Trabalho
    git add .

    # Salvar as alterações no repositório local
    git commit -m "comentário sobre o que foi feito"

Enviando as alterações para o repositório remoto

  • Depois que nossas alterações foram registradas no repositório local, precisamos enviar para o repositório remoto.(github.com)
  • Para isso usamos o comando git push. 

    # envia as alterações as alterações locais para o repositório remoto
    # normalmente hospedado no site github.com
    git push

    # se o repositório não existe na nuvem temos que forçar o seu envio
    git remote add origin https://endereco/do/servidor

Criando ramificações - branches

  • Podemos criar branchs para nosso repositório.
  • Assim podemos criar uma nova versão de nosso código sem interferir no que já está pronto ou o que está sendo desenvolvido por outra pessoa.
  • Para isso usamos o comando git branch.
  • Podemos também alternar entre as branch com o comando git checkout

    # cria uma nova branch do código para um novo desenvolvimento
    git branch nome-da-branch

    # alternando para a branch criada
    git checkout nome-da-branch

    # enviando a branch para o repositório remoto
    git push origin nome-da-branch

Atualizando e mesclando branch

  • Podemos buscar as alterações mais recentes em uma branch usando o comando git pull
  • Usando o comando git merge branch-name podemos mesclar a branch atual com a branch branch-name
  • Podemos fazer uma previsão de conflito entre as branchs se houver usando o comando git diff 

    # busca os códigos atuais no repositório da nuvem
    git pull origin 'branch_name'

    # mesclando as branchs
    git merge 'branch_name'

    # antes de mesclar um branch podemos fazer uma previsão
    # se haverá conflitos entre as branchs
    git diff branch-que-sera-mesclado branch-onde-sera-mesclado

Resetando uma branch

  • Podemos começar de novo com a versão mais recente do código do repositório remoto zerando a branch desejada.
  • Para isso usamos o comando git reset --hard. 

    # Descarte todas as minhas alterações locais e faça meu ramo atual
    # ser idêntico ao ramo do repositório remoto
    git reset --hard origin/master

Renomeando uma branch

  • Podemos renomear uma branch usando o código abaixo. 

    # Renomeando uma branch
    git branch -m master main

Criar um repositório privado (onedrive)

  • Podemos criar um repositório privado para o onedrive.
  • Assim não teremos as limitações que o github impõe ( arquivos de no máximo 100mb / 2Gb por repositório )

    # Vá até a pasta que manterá os arquivos do Git
    cd /caminho/para/a/pasta/que/manterá/os/arquivos/no/onedrive
    # inicie o repositório na pasta atual
    git init
    # criando o repositório Bare. Um repositório bare
    # no Git é aquele que não possui um diretório de trabalho.
    git config --bool core.bare true

    # Vá ao diretório de trabalho e clone o repositório Bare criado
    git clone /caminho/para/a/pasta/que/manterá/os/arquivos/no/onedrive

Não monitorar certos aquivos e pastas



    # ignorar tudo (dirs, files, sub-dirs, sub-files)
    /*
    # não ignorar o .gitignore
    !.gitignore

    # não ignorar o diretório  dir
    !/fish

    # ignorar os arquivos e subdiretorios /fish
    /fish/*

    # não ignorar o arquivo config.fish
    !/fish/config.fish
