História

  • projetada por Rasmus Lerdorf na década de 1990, para contar as visitas que eram em seu currículo on-line.
  • O PHP é uma linguagem popular de script (scripting language) de propósito geral
  • É especialmente adequada para desenvolvimento web, pois alteramos o HTML de forma dinâmica lá no lado do servidor.
  • Rápido, flexível e pragmático

Evolução

  • A versão PHP 7 trouxe um maior amadurecimento ao código php, e cada vez mais ele está caminhando para uma Orientação à Objetos mais completa, mas ainda podendo ser usado com os conceitos mais antigos.
  • Sempre cheque qual a versão instalada em sua hospedagem, pois existem diferenças entre o que se pode ou não usar.
  • Vem ganando novos recursos: autoload, tipagem de variáveis (7.4+), etc

Tipos de Linguagens

  • Existem duas categorias principais:
  • Linguagens Compiladas ( Java, C# ): o código é traduzido para linguagem de máquina antes da execução. (.apk, .exe)
  • Linguagens Interpretadas ou processadas (PHP, Python, Javascript): o código é executado linha por linha por um interpretador.

Paradigmas de Programação

  • Existem diversos paradigmas.
  • Um paradigma é uma forma ou norma para se fazer algo.
  • No PHP podemos usar várias formas, em linguagens como Java apenas a POO
  • Procedural ou Estruturada: Estrutura o código em procedimentos ou funções.
  • Orientado a Objetos: Utiliza classes e objetos para organizar o código.

Programação Orientada à Objetos

  • Pensar em programação orientada a objetos é como organizar um jogo de Lego.
  • Cada bloco (objeto) tem suas próprias peças (dados) e habilidades (funções).
  • Você cria diferentes tipos de blocos (classes) com características específicas, e, quando necessário, os combina para construir algo maior (programa).
  • Isso ajuda a estruturar o código de forma mais modular e compreensível para humanos e máquinas.
  • Nasceu em 1960 e ganhou força com linguagens como JAVA, C#, Python e desde PHP7.

Classes

  • Uma classe em programação é como um modelo ou um plano para criar objetos.
  • Pense nela como um esboço que define as características e comportamentos que os objetos criados a partir dela terão.
  • Possui 04 pilares:
  • Encapsulamento: Consiste em agrupar dados (atributos) e métodos (comportamentos) relacionados em uma unidade chamada classe protegendo/limitando seus acessos.

  • Herança: Permite que uma classe herde características (atributos e métodos) de outra classe. A classe que é herdada é chamada de classe pai ou superclasse, e a classe que herda é chamada de classe filha ou subclasse.
  • Polimorfismo: Permite que objetos de diferentes classes sejam tratados como objetos de uma mesma classe por meio da sobreposição de métodos simplifiando a interação entre classes.

  • Abstração:Envolve a criação de classes e objetos que representam entidades do mundo real, simplificando e abstraindo os detalhes complexos.
            
            
                class Animal {
                    private $nome;
                    private $som;
                
                    public function __construct($nome, $som) {
                        $this->nome = $nome;
                        $this->som = $som;
                    }
                
                    public function fazerSom() {
                        echo "{$this->nome} faz {$this->som}\n";
                    }
                }
                
                // Criando instâncias da classe Animal
                $gato = new Animal("Gato", "miau");
                $cao = new Animal("Cachorro", "auau");
                
                // Chamando método da classe
                $gato->fazerSom();
                $cao->fazerSom();

Tipos de Acesso no PHP

  • Public: são acessíveis de qualquer lugar, dentro ou fora da classe. Isso significa que eles podem ser acessados diretamente, sem restrições.
  • Private: são acessíveis apenas dentro da própria classe. Isso impede que sejam acessados ou modificados por código fora da classe.
  • Protected:são semelhantes aos private, mas com uma exceção: eles também podem ser acessados por classes derivadas (subclasses).

Abstração | Polimorfismo

            
                
                    abstract class Animal {
                        abstract public function emitirSom();
                    }
                    
                    class Cachorro extends Animal {
                        public function emitirSom() {
                            echo "Au Au\n";
                        }
                    }
                    
                    class Gato extends Animal {
                        public function emitirSom() {
                            echo "Miau\n";
                        }
                    }
                    
                    $cachorro = new Cachorro();
                    $gato = new Gato();
                    
                    $cachorro->emitirSom();
                    $gato->emitirSom();   
                
        
            
                
                    interface EmitirSom {
                        public function som();
                    }
                    
                    class Cachorro implements EmitirSom {
                        public function som() {
                            echo "Au Au\n";
                        }
                    }
                    
                    class Pato implements EmitirSom {
                        public function som() {
                            echo "Quack Quack\n";
                        }
                    }
                    
                    $cachorro = new Cachorro();
                    $pato = new Pato();
                    
                    function executarSom(EmitirSom $animal) {
                        $animal->som();
                    }
                    
                    executarSom($cachorro);
                    executarSom($pato);
